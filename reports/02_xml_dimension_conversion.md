# Pattern-B パーツXML 寸法変換レポート

**対象**: `models/parts/servo_x.xml`, `servo_y.xml`, `bracket.xml`, `camera.xml`
**作成日**: 2026-07-04
**位置づけ**: 実測/仕様値の寸法(mm)を、MJCFの`<geom>`要素にどう変換したかを記録する。servo_x/servo_yは同一寸法のため1セクションに統合。

---

## MJCF geomサイズの基本ルール(前提)

`type="box"`の`size`属性は**全長ではなく半径(half-extent)**を指定する。
すなわち実測の全長(mm)を1000で割ってメートル化し、さらに**2で割った値**を`size`に書く。

```
size_x = (実測の全長_mm / 1000) / 2
```

以下、この変換を各パーツに適用する。

---

## 1. サーボ (servo_x.xml / servo_y.xml) — MG996R

### 実測/仕様値
| 項目 | 値 | 出典 |
|---|---|---|
| 寸法 | 40.7 x 19.7 x 42.9 mm | 標準データシート(確定値) |
| 質量 | 55g | 標準データシート(確定値) |

### 変換
| 実測全長(mm) | 半径換算(m) | XML size値 |
|---|---|---|
| 40.7 (X) | 40.7/1000/2 = 0.02035 | 0.0204 (四捨五入) |
| 19.7 (Y) | 19.7/1000/2 = 0.00985 | 0.0099 |
| 42.9 (Z) | 42.9/1000/2 = 0.02145 | 0.0215 |

質量はそのままkg換算: 55g → 0.055kg

### XML該当箇所

```xml
<!-- servo_x.xml -->
<mujocoinclude>
  <body name="servo_x_body" pos="0 0 0.0215">
    <geom name="servo_x_geom" type="box"
          size="0.0204 0.0099 0.0215"
          mass="0.055" rgba="0.2 0.2 0.25 1"/>
  </body>
</mujocoinclude>
```

```xml
<!-- servo_y.xml (寸法・質量は同一。posのみ異なる) -->
<mujocoinclude>
  <body name="servo_y_body" pos="0 0 0">
    <geom name="servo_y_geom" type="box"
          size="0.0204 0.0099 0.0215"
          mass="0.055" rgba="0.2 0.2 0.25 1"/>
  </body>
</mujocoinclude>
```

### pos属性の意味(2ファイルの違い)

- `servo_x_body`: `pos="0 0 0.0215"` — Z寸法の半径(0.0215)だけ持ち上げている。これはサーボ本体の**中心**が原点(base取り付け面)からZ方向に半分浮いた位置にあることを表す(サーボ底面がbaseに接する配置)。
- `servo_y_body`: `pos="0 0 0"` — 統合XML側の`tilt_link`原点にそのまま配置される設計のため、パーツ内では原点シフトなし。

この違いは「パーツ単体は形状のみを持ち、実際の取り付け位置は統合XML(`pattern_b_integrated.xml`)側のbody階層で決める」という設計方針の結果である(Q3で確認した設計方針)。servo_xは固定ベース直上という位置関係がパーツ単体でも自明なため先出しで加算し、servo_yはtilt_linkとの相対関係が統合側の`<body pos="0 0 0.05">`で決まるため、パーツ側は0のままにしている。

## 2. ブラケット (bracket.xml) — コの字ジンバルブラケット

### 実測/仕様値
| 項目 | 値 | 出典 |
|---|---|---|
| 標準ジンバルセット全体 | 58 x 37 x 25.5mm(代表値) | 参考商品(AliExpress)の代表寸法、assumed |
| 縦アーム高さ(PAN出力軸→TILT軸) | 50mm | ユーザー確認済み(「ジンバルブラケットが参考商品と完全に同一」) |
| 質量 | 18g | 代表値(assumed) |

### 変換方針

ブラケット全体を1つの`box`で簡略化している。実物はコの字(U字)型で複数面があるが、
Level1/2の段階では**縦アーム1本の直方体**として近似した。理由は、Level1/2の検証目的が
「静止保持」「min-max範囲でのスイープ」であり、ブラケットの詳細形状よりも
「PAN軸からTILT軸までの距離(=腕の長さ)」が力学的に重要な要素だったため。

| 項目 | 実測/代表値 | 変換 | XML size値 |
|---|---|---|---|
| 縦アーム高さ | 50mm(参考: PAN→TILT軸間) | 50/1000/2 = 0.025 | 0.025 (Z) |
| 断面(幅・厚み) | 標準ジンバル代表値から簡略化 | 16mm→半径0.008 / 25mm→半径0.0125 | 0.008 (X) / 0.0125 (Y) |

質量: 18g → 0.018kg (代表値のまま)

### XML該当箇所

```xml
<mujocoinclude>
  <body name="bracket_body" pos="0 0 0.025">
    <geom name="bracket_geom" type="box"
          size="0.008 0.0125 0.025"
          mass="0.018" rgba="0.75 0.75 0.8 1"/>
  </body>
</mujocoinclude>
```

`pos="0 0 0.025"`は、ブラケット中心をZ方向に半径分(0.025)持ち上げている。
これはbracket_bodyが`pan_link`原点(PAN軸位置)から真上に伸びる形状であることを表し、
統合XML側で`tilt_link`をさらに`pos="0 0 0.05"`(=ブラケット全長)だけ上に配置することで
「PAN軸からTILT軸まで50mm」という実測値を再現している。

**精度上の注意**: X/Y方向のsize(0.008, 0.0125)は標準ジンバルセットの代表値からの簡略化であり、
実測値ではない(assumed)。今後実測すればここを更新する。

## 3. カメラ (camera.xml) — RGBウェブカメラ

### 実測値
| 項目 | 値 | 出典 |
|---|---|---|
| 寸法 | 70 x 30 x 30mm | ユーザー実測(2026-07-03、確定値) |
| 質量 | 50g | ユーザー実測(確定値) |
| 筐体材質 | プ�ラスチック | ユーザー確認済み |

### 変換
| 実測全長(mm) | 半径換算(m) | XML size値 |
|---|---|---|
| 70 (X, 長さ方向) | 70/1000/2 = 0.035 | 0.035 |
| 30 (Y, 幅) | 30/1000/2 = 0.015 | 0.015 |
| 30 (Z, 高さ) | 30/1000/2 = 0.015 | 0.015 |

質量: 50g → 0.050kg (実測値そのまま)

### XML該当箇所

```xml
<mujocoinclude>
  <body name="camera_body" pos="0.055 0 0">
    <geom name="camera_geom" type="box"
          size="0.035 0.015 0.015"
          mass="0.050" rgba="0.15 0.65 0.35 1"/>
  </body>
</mujocoinclude>
```

### pos属性の意味 — TILT軸オフセットの再現

`pos="0.055 0 0"`のX方向0.055m(55mm)は、実測寸法そのものではなく、
**TILT軸からカメラ重心までのオフセット**をユーザーが以下のように算出した値である。

```
ブラケットのカメラ接着面までの長さ  = 40mm
カメラ本体の端から中心までの長さ    = 30mm(全長70mmの半分ではなく、
                                       ユーザー算出時の代表値15mmを使用)
TILT軸からカメラ重心までの距離     = 40 + 15 = 55mm
```

これにより、camera_bodyは統合XML側の`tilt_link`原点(TILT軸そのもの)から
X方向に55mm離れた位置に配置され、「TILT軸から片持ちで55mm先にカメラがある」
という実機の片持ち配置を再現している。

**精度上の注意**: この55mmは「商品ページと記憶からの算出」(ユーザー申告)であり、
実測ではない(assumed)。ブラケット接着面の40mmとカメラ半長の15mmは、
今後実測で精度向上の余地がある。

## 4. 統合XML側でのbody階層と実測値の対応(全体像)

パーツ単体では表現できない「パーツ間の位置関係」は統合XML(`pattern_b_integrated.xml`)側の
`<body pos="...">`で表現している。実測値との対応を以下にまとめる。

| 統合XML内のpos | 値 | 対応する実測/仕様値 |
|---|---|---|
| `base`内`servo_x`のinclude位置 | (servo_x.xml内で自己完結) | MG996R寸法 |
| `pan_link pos="0 0 0.043"` | 43mm | PANサーボ高さ(42.9mm)相当、出力軸位置 |
| `tilt_link pos="0 0 0.05"` | 50mm | ブラケット縦アーム高さ(ユーザー確認済み) |
| `camera_body pos="0.055 0 0"` | 55mm | TILT軸-カメラ重心オフセット(ユーザー算出) |

この構造により、「実測値の変更 → 対応するpos/sizeを1箇所書き換えるだけで済む」
トレーサビリティを確保している。
