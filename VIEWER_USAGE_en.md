# PX MuJoCo model: 3D / 360-degree viewing

Run commands from the `px_sim_v4` directory.

## 1. Install

```bash
python -m pip install -r requirements.txt
```

## 2. Fastest manual 3D view

Linux / Windows:

```bash
python -m mujoco.viewer --mjcf=models/pattern_b_integrated.xml
```

macOS:

```bash
mjpython -m mujoco.viewer --mjcf=models/pattern_b_integrated.xml
```

Use the MuJoCo viewer mouse controls to orbit, pan and zoom around the model.

## 3. Automatic 360-degree orbit

Linux / Windows:

```bash
python viewer/view_model_360.py --auto
```

macOS:

```bash
mjpython viewer/view_model_360.py --auto
```

Controls:

- `Space`: pause/resume automatic camera rotation
- `R`: reset the camera azimuth
- Close the window to exit

Useful adjustments:

```bash
mjpython viewer/view_model_360.py --auto \
  --speed 20 \
  --elevation -25 \
  --distance 0.30 \
  --yaw 90 \
  --pitch 90
```

The script rotates the **viewer camera** around the model. It does not rotate the physical base, so the simulated model and gravity remain unchanged.


# PX Sim v4 — Using the v3 and v4 Viewers

The original `scripts/v3/` and `scripts/v4/` directories have not been modified.

All 3D visualization code is kept separate in:

- `scripts/v3_viewer/`
- `scripts/v4_viewer/`
- `scripts/common/`

This allows the original automated tests to remain unchanged while providing separate scripts for interactive visual inspection.

## Added Directory Structure

```text
px_sim_v4/
└── scripts/
    ├── common/
    │   └── runtime_viewer.py
    ├── v3/                    # Original files, unchanged
    ├── v3_viewer/
    │   ├── optics.py          # Copied from v3
    │   ├── px_env.py          # Copied from v3; control logic unchanged
    │   ├── run_level1_static.py
    │   ├── run_level2_sweep.py
    │   └── run_level3_tracking_tests.py
    ├── v4/                    # Original files, unchanged
    └── v4_viewer/
        ├── optics.py
        ├── px_env.py
        ├── tracking_mode_fsm.py
        ├── run_level4_fsm_tests.py   # Direct copy of the original v4 test
        └── run_level4_fsm_visual.py
```

## Setup

Run the following commands from the project directory:

```bash
cd px_sim_v4
python3 -m pip install -r requirements.txt
```

On macOS, use `mjpython` instead of `python3` when opening the 3D viewer.

On Linux and Windows, use `python3` or `python`.

## Example Commands for macOS

```bash
# Level 1: View the model holding a static position
mjpython scripts/v3_viewer/run_level1_static.py --duration 5 --auto-orbit

# Level 2: View the PAN/TILT sweep
mjpython scripts/v3_viewer/run_level2_sweep.py --auto-orbit

# Level 3: Run and visualize all PD tracking tests
mjpython scripts/v3_viewer/run_level3_tracking_tests.py --test all

# Level 3: Run only the noise and detection-dropout test
mjpython scripts/v3_viewer/run_level3_tracking_tests.py --test test2

# Level 4: Run the copied FSM automated tests without a viewer
python3 scripts/v4_viewer/run_level4_fsm_tests.py

# Level 4: Visualize WATCH/DETECT state transitions in 3D
mjpython scripts/v4_viewer/run_level4_fsm_visual.py --auto-orbit
```

On Linux and Windows, replace `mjpython` with `python3` or `python`.

## Common Viewer Options

| Option | Description |
|---|---|
| `--auto-orbit` | Automatically rotate the camera 360 degrees around the model |
| `--orbit-speed 20` | Set the camera rotation speed in degrees per second |
| `--render-fps 60` | Set the viewer refresh rate |
| `--playback-speed 0.5` | Play the simulation at half speed |
| `--no-realtime` | Disable real-time synchronization and run as fast as possible |
| `--keep-open` | Keep the viewer open after the scenario finishes |
| `--hide-target` | Hide the red target-position marker |
| `--hide-debug` | Hide the debug information shown in the upper-left corner |
| `--headless` | Run without opening the viewer; useful for CI or validation |

## Keyboard Controls

| Key | Action |
|---|---|
| `Space` | Pause or resume the simulation display |
| `O` | Pause or resume automatic 360-degree camera rotation |
| `R` | Reset the camera to its initial position |

Standard MuJoCo mouse controls are also available:

- Rotate the camera
- Pan the camera
- Zoom in and out

## Output Files

To avoid overwriting the original results, plots generated by the viewer scripts are saved under:

```text
outputs/viewer/
├── level1_static_viewer.png
├── level2_sweep_viewer.png
└── level3_tracking_tests.png
```

## Design Separation

The viewer implementation is intentionally separated from the original simulation and test logic.

- `scripts/v3/` and `scripts/v4/` contain the original automated tests.
- `scripts/v3_viewer/` is used for visual inspection of v3 physical motion and PD tracking.
- `scripts/v4_viewer/run_level4_fsm_tests.py` is a copy of the original FSM test script.
- `scripts/v4_viewer/run_level4_fsm_visual.py` plays the FSM transitions slowly enough for a person to inspect them.
- `scripts/common/runtime_viewer.py` provides the shared viewer, real-time synchronization, automatic camera orbit, and debug display.

The viewer code is implemented as a separate presentation layer. It does not modify the XML model, PD controller, or FSM logic.
